# 设计说明

## 模块设计

Moon PathPlanning 用 MoonBit 包边界组织可复用能力，而不是把算法写成单文件
脚本。

- `src/core` 放置坐标、搜索状态、错误和统一结果。
- `src/grid` 管理网格尺寸、障碍物、障碍物膨胀、地形代价和移动模式。
- `src/graph` 提供与网格解耦的加权邻接图。
- `src/heuristics` 提供 A 星族算法可选启发函数。
- `src/algorithms` 实现 BFS、DFS、Dijkstra、A 星和双向 A 星。
- `src/region` 提供论文 RS-APSO 方向所需的障碍物边界角识别和候选搜索区域生成。
- `src/swarm` 提供 RS-APSO 所需的适应度、随机源、自适应参数和基础 PSO/RS-APSO 主循环。
- `src/continuous` 提供栅格中心线段栅格化、静态线段可见性和路径快捷平滑，作为后续采样规划与连续空间后处理的几何基础。
- `src/dynamic` 提供动态障碍物碰撞半径、碰撞检测、速度方向预测、边界往复预测、连续坐标时间预测、连续轨迹安全评估、连续碰撞诊断报告、连续轨迹最小安全间距评估、连续安全感知动态避障、连续等待避障和跳跃避障路径修正。
- `src/planner` 统一调度经典搜索、基础 PSO 与 RS-APSO，向调用方返回同一种结果结构。
- `src/io` 与 `src/visualize` 处理示例 JSON、JSON 字符串解析、路径 SVG 和区域搜索 SVG 输出边界。

## 数据结构设计

`Point` 是路径算法共享的二维整数坐标。`GridMap` 持有起点、终点、障碍物和
目标格地形代价；算法通过网格邻居接口获得可扩展节点，不直接操作存储细节。
`inflate_obstacles(radius)` 使用 Chebyshev 半径扩展障碍物，为机器人尺寸和安全边距
预留建模入口。
`PathResult` 同时包含路径、总代价、搜索状态、访问节点数和展开节点数，方便
测试、CLI 和可视化复用。

`WeightedGraph` 使用节点数组和边数组维护邻接关系。初版实现偏向可读性和确定性，
后续在 benchmark 中观察热点后再决定是否引入更专门的数据结构。

## 算法选择原因

BFS 和 DFS 给出无权搜索的基础参照；Dijkstra 是权重图最短路基线；A 星通过启发
函数减少无关扩展；双向 A 星用于展示从两端收缩搜索空间的工程接口。初版优先
覆盖网格地图的确定性行为，便于在测试里比较路径成本和搜索状态。

## MoonBit 化改写说明

参考项目 `zhm-real/PathPlanning` 提供了路径规划领域的算法主题与演示方向。本项目
保留算法思想归属说明，但源码按 MoonBit 的模块、包和类型系统重新组织。网格模型、
统一结果、Planner、测试和 SVG 字符串输出都在库边界内建模，调用方不需要依赖
Python 脚本目录约定。
