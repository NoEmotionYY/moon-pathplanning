# 路线图

## 第一阶段：基础工程和搜索算法

- 建立 MoonBit 项目结构、执行日志、测试目录与许可证。
- 实现网格模型、图结构、启发函数、BFS、DFS、Dijkstra、A 星和双向 A 星。
- 提供统一 Planner 返回结构。

## 第二阶段：JSON、CLI、SVG、benchmark

- 固化示例地图 schema 与基础 JSON 序列化。
- 提供演示 CLI 和 SVG 路径导出。
- 记录 benchmark 场景并为热点数据结构建立性能基线。

## 第三阶段：区域搜索与 RS-APSO

- 根据《基于区域搜索粒子群算法的机器人路径规划》整理区域搜索预处理、RS-APSO 与动态避障开发边界。
- 新增障碍物膨胀安全边距，支持地图预处理阶段保守建模。
- 新增障碍物边界角识别、候选搜索区域生成和区域搜索测试。
- 新增 swarm 基础层，覆盖路径适应度、固定 seed 随机源、PSO 参数默认值、基础 PSO 和 RS-APSO 路径生成。
- 新增 dynamic 基础层，覆盖碰撞半径、碰撞判断、速度方向预测和跳跃避障路径修正。
- 新增区域搜索 SVG 叠加导出，便于观察候选区域与边界角。
- 新增 20x20 简单/复杂 RS-APSO benchmark 示例地图。
- 将基础 PSO 与 RS-APSO 接入 Planner 统一调度。
- 新增初始 RS-APSO benchmark runner，输出经典算法、PSO 与 RS-APSO 的 CSV 指标。
- 新增 benchmark 默认 5 次重复运行与总/平均耗时统计。
- 新增 JSON 地图字符串解析入口，支持 v1 schema 的 `format`、尺寸、起终点、移动模式、障碍物和 terrain。
- 新增边界往复动态障碍物的可复现场景和跳跃避障测试。
- 新增 native 后端 JSON 文件型 CLI 输入，复用 JSON v1 字符串解析入口运行 A 星 demo。
- 新增 CLI 与 benchmark JSON 字符串输入参数，提供不依赖文件系统的跨后端输入路径。
- 新增嵌入式示例地图入口，CLI 与 benchmark 可通过 `--example/-e` 在非 native 后端复用固定 JSON v1 示例。
- 新增连续坐标动态障碍物模型，支持 Double 坐标、按时间步预测、连续碰撞判断和跳跃修正。
- 新增连续轨迹安全评估，支持带时间线段采样并检测连续动态障碍物碰撞风险。
- 新增连续碰撞诊断报告，支持定位首个碰撞线段、采样点、障碍物索引、碰撞时间和双方连续坐标。
- 新增连续轨迹最小安全间距评估，支持返回最接近采样点的距离、碰撞半径和净安全间距。
- 新增动态避障 benchmark 场景，覆盖静态 A 星基线、整数速度修正、边界往复修正和连续坐标时间步修正。
- 新增连续安全感知动态避障，按连续候选线段的最小安全间距选择横向跳点。
- 新增连续等待动态避障，支持狭窄通道中通过原地等待避开连续动态障碍物。
- 新增 benchmark JSON 文件输入参数，native 后端可复用 JSON v1 地图文件输出同格式 CSV 指标。
- 新增 10x10 多方向穿越动态 benchmark 场景，覆盖多个动态障碍物对静态路径的修正影响。
- 新增动态 benchmark 连续安全指标，输出连续动态行是否安全以及最小安全间距。
- 新增 12x12 混合动态 benchmark 场景，覆盖静态障碍、三方向动态障碍物、边界往复、连续安全感知和连续等待修正。
- 继续完善更高级的连续空间规划模型。

## 第四阶段：增量规划与采样规划

- 引入 LPA* 与 D* Lite 处理地图增量变化。
- 评估 RRT、RRT-Connect 与 RRT* 在连续空间中的建模边界。
- 补充更丰富的障碍物、采样与可视化案例。

## 第五阶段：mooncakes.io 发布与生态适配

- 收敛公开 API 与版本策略。
- 补充文档示例、兼容性说明和发布校验。
- 适配 MoonBit 生态中的依赖管理与包发布流程。
